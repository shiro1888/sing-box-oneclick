# 这个项目到底怎么实现的（原理说明）

一句话：**一个 bash 脚本，在你的 VPS 上把「代理核心 + 订阅分发 + 流量限额 + 网络调优」一次性装好并接成一套**。客户端拉一个订阅链接就能用。

下面从「整体架构 → 一次安装做了什么 → 每个机制的原理 → 安全设计」讲清楚。

---

## 0. 整体架构（一张图）

```
                    你的 VPS（Debian/Ubuntu/RHEL + systemd）
   ┌──────────────────────────────────────────────────────────────┐
   │  sing-box（代理核心，一个进程，5 个入站 inbound 同时监听）        │
   │    ├─ Hysteria2   UDP   ← QUIC，抗丢包，主力                     │
   │    ├─ AnyTLS      TCP   ← sing-box ≥1.12 才有                    │
   │    ├─ VLESS-Reality TCP ← “偷”大网站证书，最隐蔽                 │
   │    ├─ Shadowsocks-2022 TCP+UDP ← 老牌兜底                        │
   │    └─ CF-Vless（可选）127.0.0.1 ← 只给 cloudflared 隧道用        │
   │  route 段：禁 BT/PT + 拦广告 + （可选）解锁站点走 WARP 出站        │
   ├──────────────────────────────────────────────────────────────┤
   │  nginx（:80）  → 在随机路径发布 3 种订阅 + 可视化看板页           │
   │  vnstat + python（cron 每5分钟）→ 统计本月流量、写限额头、超额停机 │
   │  nftables      → HY2 端口跳跃（一段 UDP 端口重定向到真实端口）      │
   │  sysctl        → 网络调优（16MB UDP 缓冲 + BBR）                  │
   └──────────────────────────────────────────────────────────────┘
            ↑ 客户端（Clash/Mihomo、v2rayN、Shadowrocket、SFA…）
              拉订阅 URL → 拿到全部节点 → 选一个连
```

核心思路：**sing-box 负责“代理”，nginx 负责“把配置发给客户端”，其余都是围绕这两个的运维件**。

## 1. 一次 `install` 到底按什么顺序做了什么（`do_install`）

脚本是**幂等**的：密钥已存在就复用、配置每次重渲染。流水线顺序（顺序本身有讲究）：

| # | 步骤 | 干什么 / 为什么这个顺序 |
|---|------|------|
| 1 | `umask 077` | 之后所有新建文件默认 600/700，消除“先明文写、后 chmod”的可读时间窗 |
| 2 | `validate_inputs` | 端口/SNI/到期时间不合法**立刻报错**，不会装到一半才崩 |
| 3 | `detect_os` | 认 apt/dnf/yum，强制要求 systemd |
| 4 | `install_deps` | 装 curl/nginx/vnstat/openssl/cron/python3/qrencode… |
| 5 | `time_sync` | 开 NTP——**Reality/TLS 对时钟敏感**，时间不准会“连得上但代理不通” |
| 6 | `install_singbox` | 跑官方脚本装核；读版本号决定 AnyTLS 是否可用（<1.12 自动跳过） |
| 7 | `detect_net` | 探公网 IP + 默认网卡（流量统计要按网卡算） |
| 8 | `check_reality_sni` | 探一下偷证书目标是否真支持 TLS1.3+H2，填错给提示 |
| 9 | `gen_secrets` | 生成/复用所有密钥（HY2/AnyTLS/UUID/Reality 密钥对/SS/obfs）→ 写 `secrets`(600) |
| 10 | 载入 `cf.env`/`warp.env` | 接过 CF/WARP 的话，重装时保留 |
| 11 | `gen_cert` | 自签 TLS 证书（HY2/AnyTLS 用） |
| 12 | `config_sysctl` | **在 sing-box 启动前**调好内核缓冲，这样 HY2 一启动就拿到大 UDP 缓冲 |
| 13 | `write_env` | 把运行参数（限额/到期/网卡/计费/HY2进阶/路由开关）写 `/etc/sing-box-node.env` |
| 14 | `write_singbox_config` | 渲染 `config.json` → **`sing-box check` 校验通过才启动**（不过直接报错） |
| 15 | `write_subscription` | 生成 Clash订阅 + 通用base64订阅 + 看板页到 nginx 根目录 |
| 16 | `config_nginx` | 配 nginx：随机路径 location + 流量头；`nginx -t` 通过才 reload |
| 17 | `install_traffic` | 装 python 限额脚本 + 每5分钟 cron |
| 18 | `config_firewall` | （提示云安全组）可选放行 ufw |
| 19 | `config_porthop` | （若开）装 nftables 端口跳跃 |
| 20 | `print_summary` | 打印订阅 URL、节点、管理命令、需手动完成的事 |

## 2. 五个协议节点：各自原理 + 防什么

为什么是“多协议并存”？**不同协议在不同网络环境下生存能力不同**，一个被封/被限速还有别的能用。

### ① Hysteria2（主力，UDP/QUIC）
- 基于 **QUIC over UDP**，自带前向纠错，**丢包率高的跨境线路上比 TCP 类协议快得多**。
- 用**自签证书** + 客户端 `skip-cert-verify`（节点是你自己的，不需要 CA 信任链）。
- 三个进阶（都可选）：
  - **obfs（salamander）**：给 QUIC 包再裹一层混淆，抗“UDP+QUIC 指纹识别”。
  - **端口跳跃**：见第 6 节。
  - **brutal（up/down）**：客户端订阅里写死上下行带宽，让 Hysteria 用“暴力”拥塞控制顶满带宽（写在订阅 `up:/down:`，是客户端侧开关）。

### ② AnyTLS（sing-box ≥1.12）
- 把流量整成**普通 TLS** 的样子，抗“TLS 流量分析”。版本不够时脚本**自动跳过**，不报错。

### ③ VLESS-Reality-Vision（最隐蔽，TCP）
- **Reality = “偷证书”**：握手时不发自签证书，而是**实时把你指定的大网站（默认 `www.bing.com`）的真证书借过来用**。
  - 关键配置：`server_name` 和 `handshake.server` 都指向那个大站，`server_port: 443`。
  - 对审查者来说，这条连接**看起来就是在访问 bing.com 的正常 HTTPS**，没有自签证书这种破绽。
  - 所以 `check_reality_sni` 要确认那个目标**真支持 TLS1.3+H2**——目标本身不达标，伪装就露馅。
- `flow: xtls-rprx-vision` 是 XTLS 的流控，减少“TLS in TLS”的双重加密开销。

### ④ Shadowsocks-2022（兜底，TCP+UDP）
- 用 **SIP022** 新规范：分享链接是 `ss://method:密码@host:port`（密码百分号编码，**不是老的 base64**）。
- 密钥长度按加密方法自适应（128-gcm=16 字节，256/chacha=32 字节），openssl 随机 + base64。
- 老牌、客户端兼容性最好，作为“前面几个都不行时”的保底。

### ⑤ CF-Vless / “Argo 隧道”（可选第 5 节点，大保底）
- 原理：**VLESS-over-WebSocket**，sing-box 只在 `127.0.0.1` 开一个本地入站，由 **cloudflared 隧道**把 Cloudflare 边缘的流量转进来。
- 意义：**连你 VPS 的 IP 被墙了，照样能用**——因为客户端连的是 Cloudflare 的 IP，不是你的 VPS。
- 为什么不默认装：要你在 Cloudflare 后台建 Tunnel 拿 token，**脚本替不了你点那一步**，所以做成 `install.sh cf` 半自动。

## 3. 订阅是怎么生成和分发的

“订阅”= 把上面 5 个节点的连接信息打包成客户端能一键导入的格式。脚本出 **3 种**，覆盖几乎所有客户端：

| 格式 | 函数 | 给谁用 | 本质 |
|------|------|--------|------|
| Clash/Mihomo YAML | `render_subscription_yaml` | Clash 系 | 一份 YAML，含 proxies + 分流规则（国内直连、节点选择 MATCH） |
| 通用 base64 | `render_share_links` → base64 | v2rayN/Shadowrocket/NekoBox | 每节点一行分享链接（`hysteria2://`/`vless://`/`ss://`…），整体再 base64 |
| 每节点分享链接 | `render_share_links` | 手动单条粘贴 | 上面那批链接的明文 |

**分发机制（nginx）**：
- 订阅放在 nginx 根目录，**路径是随机的**（`/sub-<随机hex>.yaml`），等于一个“不公开就猜不到”的密钥。
- nginx 拆**两个 server**：一个 `default_server`（`server_name _`）把所有 Host 不对 / 直接打 IP 的请求一律 **404**；另一个精确绑你的 IP/域名，只在随机路径上输出订阅。脚本会先清掉自带的 `sites-enabled/default`；若机器上仍有别的 `default_server`，`nginx -t` 会报 `duplicate default server`，需手动移除冲突站点。
- 首页 `/` 直接 404，只有那几个随机路径能访问。
- Clash 订阅 location 里 `include` 一个**流量头片段**（见下一节），客户端导入后就能显示流量/到期。

**分流规则的设计**（Clash YAML 里）：你的 VPS IP、私有网段、一大票国内大厂域名（QQ/微信/B站/淘宝/百度…）、`GEOSITE,cn`/`GEOIP,CN` 全部 `DIRECT` 直连，其余 `MATCH` 走节点。**国内流量不绕代理，省流量也更快**。

## 4. 流量统计 + 自动限额（`install_traffic` 的 python 脚本）

这是“机场式”限额的核心，全靠 **vnstat（按网卡统计流量）+ 一个每 5 分钟跑一次的 python 脚本**：

1. **读数**：`vnstat --json` 拿到本月该网卡的 rx/tx 字节；按 `COUNT_MODE`（`rx+tx`/`tx`/`max`）算“已用”。
2. **写订阅头**：把 `已用/限额/到期` 写成 nginx 的 `Subscription-Userinfo` 响应头片段；变了才 reload nginx。
   - 这就是为什么**客户端能显示用了多少、还剩多少、哪天到期**——它读的就是这个头。
   - 看板页的“已用”也是用 JS 拉这个头实时显示的。
3. **限额执行**（`decide_enforcement`）：
   - 已用 ≥ 限额 → `systemctl stop sing-box` 停机，并打一个持久标志文件 `/var/lib/sing-box-node/quota-stopped`。
   - 下个月 vnstat 归零、已用 < 限额 → 自动 `systemctl start sing-box` 恢复，删标志文件。
   - 手动 `systemctl stop sing-box`（没有配额标记）不会被定时任务自动拉起；如果想永久停用，执行 `systemctl disable --now sing-box && rm -f /var/lib/sing-box-node/quota-stopped`。
   - **纯靠流量数字开关 sing-box 进程，不动配置**，简单可靠。

## 5. 网络调优（`config_sysctl`）—— 为什么这步对 HY2 很关键

写一份 `/etc/sysctl.d/99-singbox.conf`，重点两条：

- **`net.core.rmem_max/wmem_max = 16MB`**：Hysteria2 是 QUIC（UDP），**UDP 缓冲太小会被内核丢包并限速**（还会刷 quic-go 缓冲告警）。16MB 是 Hysteria 官方推荐值，**这是 HY2 跑满带宽的前提**，所以放在 sing-box 启动前设好。
- **BBR + fq**：`tcp_congestion_control=bbr` + `default_qdisc=fq`，改善 TCP 类节点（AnyTLS/Vless）的跨境吞吐。只作用于 TCP，HY2 走 UDP 不受它影响。
- 其余：TCP 大缓冲（适配高带宽-延迟积的长距离链路）、`mtu_probing`（隧道 MTU 黑洞探测）、`slow_start_after_idle=0`（代理常空闲后突发，别重置拥塞窗口）、`tcp_fastopen`、`tw_reuse`。

`doctor` 子命令会回头核对这些是否真的生效。

## 6. HY2 端口跳跃（`config_porthop`）—— 抗按端口限速

- **问题**：有些运营商专门针对“某个固定 UDP 端口”做 QoS/限速。
- **做法**：用 **nftables 在 NAT prerouting 钩子**，把**一整段 UDP 端口**（如 `20000-21000`）全部 `redirect` 到真实的 HY2 端口。
  ```
  iifname "eth0" udp dport 20000-21000 redirect to :HY2真实端口
  ```
- **效果**：客户端可以在这一段里随便挑/轮换端口连，运营商盯不住单个端口。客户端订阅里把端口段写进 `ports:`/authority，自动支持。
- 用一个 **systemd oneshot 服务**在开机时重新下发 nft 规则（重启不丢）。
- ⚠️ 代价：**云安全组要放行整段 UDP**，不只是单个端口（脚本会提示）。

## 7. 可选增强

- **WARP 解锁分流（`warp`）**：很多机房 IP 被 ChatGPT/Claude/Gemini/流媒体拉黑。原理是 `wgcf` 注册一个免费 Cloudflare WARP 账号 → 在 sing-box 里加一个 **WireGuard 出站（endpoint，tag=warp）** → route 段默认把 `geosite-openai/anthropic/google-gemini/netflix/disney`（可用 `WARP_SITES` 自定义）**指向 warp 出站**，其余仍直连。带护栏：新配置先渲染到临时文件、`sing-box check` 过了才替换，**不过就回滚，不弄坏现有节点**。
- **SSH 加固（`harden`）**：仅密钥登录 + 禁密码 + fail2ban。护栏：**必须先有授权公钥才动手**（防锁死）、用 `00-` drop-in 盖过 cloud-init 的 `PasswordAuthentication yes`、`sshd -t` 不过自动回滚。
- **可视化看板页**：一个**自包含、只读**的 HTML（订阅复制/扫码/⚡一键导入深链/每节点卡片/流量到期/延迟自测/明暗主题）。只读，不做服务器管理。
- **网页管理面板（`admin`）**：要在网页上改限额/到期/重启/备份时用。原理：一个 **python 标准库后端**，**只绑 `127.0.0.1`**（不暴露公网，只能经 SSH 隧道 `ssh -L 8088:127.0.0.1:8088` 访问）+ **随机 token**（hmac 常数时间比对）+ **白名单动作**（调用复用 `install.sh`，`subprocess` 走参数数组，无 shell 注入）。systemd 托管。这样既有网页管理，又**零公网攻击面**。
- **Komari 探针**：跟代理无关，给监控面板上报本机状态的 agent，傻瓜式装。
- **备份迁移（`backup`/`restore`）**：把密钥+参数+证书打成一个包，新机 `restore` 用**同一套凭证**重建，客户端不用换密码。
- **doctor 自检**：一次性核对服务/配置/sysctl/防火墙/安全组/时间/证书/订阅内外可达，每项给修复命令。

## 8. 贯穿全局的安全设计原则

1. **凭证最小可读**：`umask 077` 起步，`secrets`/`env` 600，`/etc/sing-box` 700。
2. **护栏式变更**：所有改 `config.json` 的操作（warp/cf）都是**先临时文件 → `sing-box check` → 通过才替换**，校验不过保留原配置。nginx 同理（`nginx -t` 过才 reload）。改 SSH 用 `sshd -t` + 自动回滚。
3. **绝不自动锁死自己**：ufw 启用前确认放行 22；harden 要求先有公钥。
4. **不公开即安全**：订阅/看板走**随机路径**；卸载时即使密钥丢了也兜底清掉所有 `sub-*/panel-*` 文件（含凭证）。
5. **明文 HTTP 的取舍**：订阅默认走 HTTP（图省事、无需域名），脚本明确警告“含全部凭证、别外传、不可信网络上 HTTPS”。
6. **用户输入净化**：进 JSON/URL/env 的用户值都清洗或转义（WARP_SITES 清成 `[a-z0-9,-]`、看板全部 `html.escape`/`json.dumps`/`urlencode`）。

## 10. 几个“为什么这么设计”的取舍

- **为什么自签证书 + skip-cert-verify**：节点是你自己的，不是要骗 CA，自签足够；省掉申域名/签证书的麻烦。Reality 是例外——它本来就不发自己的证书，而是借大站的。
- **为什么不默认开防火墙/不自动装 CF/WARP**：这些要么有锁死风险、要么需要你在外部控制台操作（CF 后台、云安全组），**脚本能做的全自动了，做不了的明确提示**，而不是假装能做。
- **为什么测试是 57 项但 WARP/harden/端口跳跃只验证“配置合法性”**：开发在 Windows 上，跑不了 `systemctl`/`sing-box`/`nft`。所以脚本里到处用**渲染函数（纯输出）+ JSON/YAML 解析断言**保证“生成的东西一定合法”，真正的运行时由真机上的 `sing-box check` 和 `doctor` 兜底。
